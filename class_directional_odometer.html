Detailed Description

A class to represent directional Odometers.

These sensors typically expose three signals lines, as opposed to two in the directionless counterparts. Depending on the state of this third signal, the direction of movement can be inferred.

overridevirtual

Initializes the Odometer to receive pulses from the specified pin.

Parameters
pinThe pin to receive pulses from, which must support external hardware interrupts
callbackThe function to be run on each incoming pulse. Please use the following as an argument: [](){yourOdometerInstance.update();}
Returns
false if an error occurred, i.e. invalid pin

You must run this function before making any calls to your Odometer or you will not get any usable distance or speed from it.

Example:

const unsigned short LEFT_ODOMETER_PIN = 2;
const unsigned long PULSES_PER_METER = 80;
DirectionlessOdometer leftOdometer(PULSES_PER_METER);
void setup() {
leftOdometer.attach(LEFT_ODOMETER_PIN, []() {
leftOdometer.update();
});
}

void DirectionalOdometer::update ( )
overridevirtual

Conducts the distance and speed measurements.

Updates the current dt with the time difference between the last two pulses and increases the pulse counter. Do not call it directly in your sketch! Instead, pass it inside a lambda as an argument to attach. Example: ``` const int ODOMETER_PIN = 2; Odometer odometer; odometer.attach(ODOMETER_PIN, [](){odometer.update();}); ```
