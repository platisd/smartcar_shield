18 ## About the library
19 The library sports a high level API, meant to enable its users to perform tasks
20 easily with relatively few lines of code. It intends to hide implementation
21 details or low level hardware operations. The library, handles the various
22 components (i.e. the vehicle and the various sensors) as objects, therefore
23 enabling trivial control over the car's functions.
24 
25 The examples are primarily tailored to be used with the
26 [Smartcar Shield](https://github.com/platisd/smartcar_shield/tree/master/extras/eagle/smartcar_shield)
27 which was the hardware initially coupled with this software. However, by using different
28 arguments the different classes can be reused with practically any Motor and Control
29 combination common to miniature vehicles. Additionally, a plethora of hobby-grade
30 sensors are supported which potentially makes this library the only external component
31 necessary for many robotic or autonomous driving use cases, when it comes to
32 programming the microcontroller.
33 
34 ## Extending the library
35 All classes implement an interface which allow you to create library extensions
36 and give you the possibility of writing code that can be reused among different
37 platforms and hardware setups. Furthermore, despite the library written for
38 Arduino-compatible microcontrollers, potentially any microcontroller could be used
39 by providing your own implementation of the Runtime interface.
